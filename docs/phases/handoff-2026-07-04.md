## How to run

```bash
# fresh fork + deploy + seed (addresses regenerate every deploy)
bash app/scripts/dev-chain/fork.sh          # anvil mainnet-fork on :8545 (background it)
pnpm -C app chain:deploy                     # DeployAnvil + SeedAnvil + writes local-deployment.json
pnpm -C app dev                              # vite on :5173
```

**Gotchas (bit us this session):**
- **Verify types with `pnpm -C app typecheck` (`tsc -b`), NEVER `tsc --noEmit`** — `tsconfig.json` is a
  project-references stub (`files: []`) so `tsc --noEmit` checks *nothing*. `pnpm build` is the real gate.
  Strict flags on (`exactOptionalPropertyTypes`, `noUncheckedIndexedAccess`). See `40a7add`.
- **A wedged mid-broadcast anvil needs a FULL restart** (`pkill -9 -x anvil` → `fork.sh`), not a
  same-fork retry — a killed `forge` broadcast leaves anvil unable to finish; retrying just re-hangs.
- Kill anvil with `pkill -9 -x anvil` (NOT `-f anvil` — matches its own cmdline). `chain:deploy` can run
  a few minutes; wait for the completion, don't kill it early on a short block-number check.
- The Vite dev server was repeatedly SIGTERM'd (143) by the environment this session — if it drops, just
  relaunch `pnpm -C app dev`.
