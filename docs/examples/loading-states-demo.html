10. Usage Guidelines

When to use each loading pattern:

Pattern
Use Cases
Skeleton
Initial page load, discovery grids, project lists, portfolio tables, activity feeds
Spinner
Wallet connection, transaction confirmation, modal loading, general async operations
Progress Bar
IPFS uploads, multi-step forms, file processing, known-duration operations
Dots
Inline loading, subtle ongoing operations, waiting for API responses
Pulse
Live data indicators, real-time price updates, active polling status
Button Loading
Any button that triggers async operations (mint, stake, claim, trade)
Scanning
Blockchain event indexing, large-scale data scanning, search operations
Shimmer
Image loading, premium content blocks, alternative to skeleton for featured content
Messages
First-time experiences, user education during loading, setting expectations
